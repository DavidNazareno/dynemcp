Welcome to DyneMCP! This guide will help you get up and running with the Model Context Protocol (MCP) framework in just a few minutes.

## What is DyneMCP?

DyneMCP is a comprehensive framework for building MCP (Model Context Protocol) servers. It provides everything you need to create, develop, and deploy MCP servers with modern TypeScript development tools.

### Key Features

- üöÄ **Fast Development**: Hot reload and instant feedback
- üèóÔ∏è **Optimized Build System**: Ultra-fast builds with esbuild
- üìù **Declarative Configuration**: Simple JSON-based configuration
- üåê **Multiple Transports**: stdio and HTTP transports
- üîß **Dynamic Registry**: Runtime registration of tools/resources/prompts
- üîí **Security Ready**: Input validation, rate limiting, and CORS

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js 18+**: [Download from nodejs.org](https://nodejs.org/)
- **pnpm**: Install with `npm install -g pnpm`

## Quick Start

### Project Creation

```bash
# Using pnpm (recommended)
pnpm create dynemcp@latest my-mcp-server

# Or install globally
pnpm add -g @dynemcp/create-dynemcp
create-dynemcp my-mcp-server
```

### Available Templates

- **default**: Basic MCP server setup (recommended for learning)
- **calculator**: Mathematical computation tools
- **http-server**: Web-based MCP server
- **secure-agent**: Production-ready with security features
- **dynamic-agent**: Advanced with learning capabilities

```bash
# Create with specific template
pnpm create dynemcp@latest my-calculator --template calculator
```

### Local Development

```bash
# Navigate to project
cd my-mcp-server

# Install dependencies (if not done during creation)
pnpm install

# Start development server
pnpm dev

# Build for production
pnpm build

# Start production server
pnpm start
```

## Creating Your First MCP Server

Let's create a simple MCP server with a basic tool. You can implement tools in two ways:

### 1. Create a New Project

```bash
pnpm create dynemcp@latest my-first-server
cd my-first-server
```

### 2. Create a Simple Tool

You can implement tools using either a class-based approach or an object-based approach:

#### Object-Based Approach (Recommended for Simplicity)

Create `src/tools/greeting.ts`:

```typescript
import { z } from 'zod'
import type { ToolDefinition } from '@dynemcp/dynemcp'

const GreeterSchema = z.object({
  name: z.string().describe('The name to greet'),
})

const greeterTool: ToolDefinition = {
  name: 'greeter',
  description: 'A simple tool that greets the user',
  schema: GreeterSchema,
  handler: async ({ name }: { name: string }) => {
    return { 
      message: `Hello, ${name}!`,
      timestamp: new Date().toISOString(),
    }
  },
}

export default greeterTool
```

#### Class-Based Approach (For More Complex Tools)

```typescript
import { DyneMCPTool } from '@dynemcp/dynemcp'
import { z } from 'zod'

export class GreetingTool extends DyneMCPTool {
  get name() {
    return 'greet'
  }

  readonly description = 'Greets a person by name'
  readonly schema = z.object({
    name: z.string().describe('Name of the person to greet'),
  })

  async execute(input: z.infer<typeof this.schema>) {
    const { name } = input
    return {
      greeting: `Hello, ${name}! Welcome to DyneMCP!`,
      timestamp: new Date().toISOString(),
    }
  }
}

export default new GreetingTool()
```

Both approaches are valid and will work with DyneMCP. Choose the one that best fits your needs:
- Use the object-based approach for simple tools with straightforward functionality
- Use the class-based approach for complex tools that need inheritance or extensive internal logic

### 3. Start the Server

```bash
pnpm dev
```

Your MCP server is now running and ready to accept requests! The tool will be automatically discovered and registered.
