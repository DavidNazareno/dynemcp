Welcome to DyneMCP! This guide will help you get up and running with the Model Context Protocol (MCP) framework in just a few minutes.

## What is DyneMCP?

DyneMCP is a comprehensive framework for building MCP (Model Context Protocol) servers. It provides everything you need to create, develop, and deploy MCP servers with modern TypeScript development tools.

### Key Features

- 🚀 **Fast Development**: Hot reload and instant feedback
- 🏗️ **Optimized Build System**: Ultra-fast builds with esbuild
- 📝 **Declarative Configuration**: Simple JSON-based configuration
- 🌐 **Multiple Transports**: stdio and HTTP transports
- 🔧 **Dynamic Registry**: Runtime registration of tools/resources/prompts
- 🔒 **Security Ready**: Input validation, rate limiting, and CORS

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js 18+**: [Download from nodejs.org](https://nodejs.org/)
- **pnpm** (recommended): `npm install -g pnpm`

## Quick Start

### Project Creation

```bash
# Using npx (recommended)
npx create-dynemcp@latest my-mcp-server

# Or install globally
npm install -g @dynemcp/create-dynemcp
create-dynemcp my-mcp-server
```

### Available Templates

- **default**: Basic MCP server setup (recommended for learning)
- **calculator**: Mathematical computation tools
- **http-server**: Web-based MCP server
- **secure-agent**: Production-ready with security features
- **dynamic-agent**: Advanced with learning capabilities

```bash
# Create with specific template
npx create-dynemcp@latest my-calculator --template calculator
```

### Local Development

```bash
# Navigate to project
cd my-mcp-server

# Install dependencies (if not done during creation)
pnpm install

# Start development server
pnpm dev

# Build for production
pnpm build

# Start production server
pnpm start
```

## Project Structure

A typical DyneMCP project looks like this:

```
my-project/
├── src/
│   ├── tools/         # Tool implementations
│   ├── resources/     # Resource definitions
│   ├── prompts/       # Prompt templates
│   └── index.ts       # Server entry point
├── dynemcp.config.json # Main configuration file
├── package.json        # Dependencies and scripts
└── tsconfig.json      # TypeScript configuration
```

## Creating Your First MCP Server

Let's create a simple MCP server with a basic tool:

### 1. Create a New Project

```bash
npx create-dynemcp@latest my-first-server
cd my-first-server
```

### 2. Create a Simple Tool

Create `src/tools/greeting.ts`:

```typescript
import { DyneMCPTool } from '@dynemcp/dynemcp'
import { z } from 'zod'

export class GreetingTool extends DyneMCPTool {
  get name() {
    return 'greet'
  }

  readonly description = 'Greets a person by name'
  readonly schema = z.object({
    name: z.string().describe('Name of the person to greet'),
  })

  async execute(input: z.infer<typeof this.schema>) {
    const { name } = input
    return {
      greeting: `Hello, ${name}! Welcome to DyneMCP!`,
      timestamp: new Date().toISOString(),
    }
  }
}

export default new GreetingTool()
```

### 3. Start the Server

```bash
pnpm dev
```

Your MCP server is now running and ready to accept requests! The tool will be automatically discovered and registered.
