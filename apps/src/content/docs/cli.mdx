DyneMCP provides two main command-line interfaces:

1. `create-dynemcp` - For creating new projects
2. `dynemcp` - For development and build commands

## Create DyneMCP CLI

The project creation CLI helps you scaffold new MCP servers.

### Installation

```bash
# Global installation (recommended)
npm install -g @dynemcp/create-dynemcp

# Or use without installing
npx @dynemcp/create-dynemcp my-project
```

### Command Syntax

```bash
create-dynemcp [directory] [options]
```

### Available Options

| Option           | Alias | Description                    | Default   |
| ---------------- | ----- | ------------------------------ | --------- |
| `--template <n>` | `-t`  | Template to use                | `default` |
| `--skip-install` |       | Skip dependency installation   | `false`   |
| `--yes`          | `-y`  | Skip all prompts, use defaults | `false`   |
| `--version`      | `-v`  | Show version number            |           |
| `--help`         | `-h`  | Show help message              |           |

### Examples

```bash
# Interactive mode with template selection
create-dynemcp my-project

# Specific template, interactive
create-dynemcp my-calculator --template calculator

# Completely non-interactive
create-dynemcp my-agent --template secure-agent --yes --skip-install

# Show available options
create-dynemcp --help
```

## DyneMCP CLI

The main DyneMCP CLI for development and build commands.

### Installation

The CLI is included when you install DyneMCP:

```bash
npm install -g @dynemcp/dynemcp
# or
pnpm add -g @dynemcp/dynemcp
```

### Development Mode

Start the server in development mode with hot reload:

```bash
dynemcp dev [options]
```

**Options**:

- `--config, -c <path>`: Path to dynemcp.config.json
- `--clean`: Clean before building
- `--transport <type>`: Override transport type (stdio, http)
- `--watch`: Enable watch mode (default: true)
- `--port <number>`: Port for HTTP transport
- `--host <string>`: Host for HTTP transport

### Production Build

Build the project for production:

```bash
dynemcp build [options]
```

**Options**:

- `--config, -c <path>`: Path to dynemcp.config.json
- `--clean`: Clean before building
- `--minify`: Minify output (default: true)
- `--sourcemap`: Generate sourcemaps (default: true)

### Watch Mode

Watch files and rebuild on changes:

```bash
dynemcp watch [options]
```

**Options**:

- `--config, -c <path>`: Path to dynemcp.config.json
- `--clean`: Clean before building
- `--transport <type>`: Override transport type

### Clean

Clean build artifacts:

```bash
dynemcp clean [options]
```

**Options**:

- `--config, -c <path>`: Path to dynemcp.config.json
- `--all`: Clean all caches and artifacts

### Analyze

Analyze the bundle size and dependencies:

```bash
dynemcp analyze [options]
```

**Options**:

- `--config, -c <path>`: Path to dynemcp.config.json
- `--json`: Output analysis as JSON
- `--detailed`: Show detailed analysis

## Configuration

The CLI reads configuration from `dynemcp.config.json` in your project root. You can override settings using command-line options.

### Example Configuration

```json
{
  "server": {
    "name": "my-mcp-server",
    "version": "1.0.0"
  },
  "tools": {
    "enabled": true,
    "directory": "./src/tools",
    "pattern": "**/*.{ts,js}"
  },
  "transport": {
    "type": "stdio"
  },
  "build": {
    "target": "node18",
    "minify": true,
    "sourcemap": true
  }
}
```

## Environment Variables

The CLI respects these environment variables:

- `DYNEMCP_CONFIG`: Path to config file
- `DYNEMCP_ENV`: Environment (development/production)
- `DYNEMCP_LOG_LEVEL`: Logging level
- `DYNEMCP_TRANSPORT`: Transport type
- `DYNEMCP_PORT`: HTTP transport port
- `DYNEMCP_HOST`: HTTP transport host

## Exit Codes

The CLI uses these exit codes:

- `0`: Success
- `1`: General error
- `2`: Configuration error
- `3`: Build error
- `4`: Runtime error

## Best Practices

1. **Development**

   - Use `dynemcp dev` for local development
   - Enable watch mode for instant feedback
   - Use sourcemaps for debugging

2. **Production**

   - Use `dynemcp build` with minification
   - Set appropriate environment variables
   - Configure logging levels

3. **CI/CD**
   - Use `--yes` flag for non-interactive mode
   - Set explicit configuration paths
   - Enable detailed error output

## Troubleshooting

Common issues and solutions:

### Build Failures

```bash
# Clean and rebuild
dynemcp clean --all
dynemcp build --clean
```

### Transport Issues

```bash
# Override transport
dynemcp dev --transport http --port 3000
```

### Configuration Problems

```bash
# Validate configuration
dynemcp build --config ./custom-config.json
```
