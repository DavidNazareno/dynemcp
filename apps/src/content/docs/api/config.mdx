# Configuration API Reference

DyneMCP uses a JSON-based configuration system that can be defined in `dynemcp.config.json` or programmatically. This guide covers all available configuration options and their usage.

## Configuration Schema

```typescript
interface DyneMCPConfig {
  server: ServerConfig
  tools: AutoloadConfig
  resources: AutoloadConfig
  prompts: AutoloadConfig
  transport?: TransportConfig
  logging?: LoggingConfig
  debug?: DebugConfig
  performance?: PerformanceConfig
  security?: SecurityConfig
  config?: ConfigOptions
}
```

## Server Configuration

```typescript
interface ServerConfig {
  name: string
  version: string
  documentationUrl?: string
}
```

Example:

```json
{
  "server": {
    "name": "my-mcp-server",
    "version": "1.0.0",
    "documentationUrl": "https://docs.example.com"
  }
}
```

## Autoload Configuration

Shared configuration for tools, resources, and prompts:

```typescript
interface AutoloadConfig {
  enabled: boolean
  directory: string
  pattern?: string
}
```

Example:

```json
{
  "tools": {
    "enabled": true,
    "directory": "./src/tools",
    "pattern": "**/*.{ts,js}"
  }
}
```

## Transport Configuration

```typescript
interface TransportConfig {
  type: 'stdio' | 'http-stream'
  options?: {
    // HTTP Stream options
    port?: number
    host?: string
    endpoint?: string
    cors?: CORSOptions
    authentication?: {
      path: string
    }
  }
}
```

Example:

```json
{
  "transport": {
    "type": "http-stream",
    "options": {
      "port": 3000,
      "host": "localhost",
      "endpoint": "/mcp",
      "authentication": {
        "path": "./src/auth.ts"
      }
    }
  }
}
```

```typescript
interface LoggingConfig {

  format: 'text' | 'json'
```

Example:

```json

  "logging": {
    "enabled": true,
    "level": "info",
    "format": "text",
    "timestamp": true,
    "colors": true
  }
}
```

```typescript
interface DebugConfig {
```

## Performance Configuration

```typescript
interface PerformanceConfig {
  maxConcurrentRequests: number
  requestTimeout: number
  memoryLimit: string
  enableMetrics: boolean
}
```

Example:

```json
{
  "performance": {
    "maxConcurrentRequests": 100,
    "requestTimeout": 30000,
    "memoryLimit": "512mb",

  }

```

## Security Configuration

```typescript
interface SecurityConfig {
  enableValidation: boolean
  strictMode: boolean
  allowedOrigins: string[]
  rateLimit: {
    enabled: boolean
    maxRequests: number
    windowMs: number
  }
}
```

Example:

```json
{
  "security": {
    "enableValidation": true,
    "strictMode": false,
    "allowedOrigins": ["*"],
    "rateLimit": {
      "enabled": false,
      "maxRequests": 100,
      "windowMs": 900000
    }
  }
}
```

## Config Options

```typescript
interface ConfigOptions {
  file?: string
  env: boolean
}
```

Example:

```json
{
  "config": {
    "file": "dynemcp.config.json",
    "env": true
  }
}
```

## Environment Variables

The following environment variables can override configuration settings:

- `DYNEMCP_CONFIG`: Path to config file
- `DYNEMCP_TRANSPORT`: Transport type
- `DYNEMCP_PORT`: HTTP Stream transport port
- `DYNEMCP_HOST`: HTTP Stream transport host

## Best Practices

1. **Minimal Configuration**
   - Only specify the options you need to override
   - Default values are sensible for most use cases

2. **Security**
   - Always enable validation in production
   - Configure CORS and rate limiting for HTTP Stream transport
   - Use authentication for production deployments

3. **Performance**
   - Adjust maxConcurrentRequests based on your server capacity
   - Set appropriate memory limits
   - Enable metrics in production for monitoring

4. **Development**
   - Enable debug mode for detailed logs
   - Use verbose logging for troubleshooting
   - Keep colors enabled for better readability

## Configuration Methods

### 1. Configuration File

Create `dynemcp.config.json` in your project root:

```json
{
  "server": {
    "name": "my-server",
    "version": "1.0.0"
  },
  "transport": {
    "type": "http-stream",
    "options": {
      "port": 3000,
      "host": "localhost",
      "endpoint": "/mcp",
      "authentication": {
        "path": "./src/auth.ts"
      }
    }
  }
}
```

### 2. Programmatic Configuration

```typescript
const server = createMCPServer()

server.config.update({
  server: {
    name: 'my-server',
    version: '1.0.0',
  },
  transport: {
    type: 'http-stream',
    options: {
      port: 3000,
      host: 'localhost',
      endpoint: '/mcp',
      authentication: {
        path: './src/auth.ts'
      }
    },
  },
})
```

### 3. Environment Variables

```typescript
// Load from .env file
import dotenv from 'dotenv'
dotenv.config()

const server = createMCPServer()
```
