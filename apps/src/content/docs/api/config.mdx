# Configuration API

Configure DyneMCP by creating a `dynemcp.config.json` (or `.ts`) file in your project root. This file controls the server name, version, and where to find your tools, resources y prompts.

## Minimal Example

```jsonc
{
  "server": { "name": "my-server", "version": "1.0.0" },
  "tools": { "enabled": true, "directory": "./src/tools" },
  "resources": { "enabled": true, "directory": "./src/resources" },
  "prompts": { "enabled": true, "directory": "./src/prompts" },
}
```

- Only include fields you need. Defaults are production-ready.
- Never expose secrets in config files.
- See the main guide for all options.
