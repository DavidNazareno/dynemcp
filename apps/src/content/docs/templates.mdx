DyneMCP provides several pre-built templates to help you get started quickly. Each template is designed for specific use cases and includes everything you need to build and deploy your MCP server.

## Using Templates

To create a new project using a template:

```bash
npx create-dynemcp@latest my-project --template <template-name>
```

Available template names: `default`, `calculator`, `http-server`, `secure-agent`, `dynamic-agent`

## Template Comparison

## Available Templates

### Default Template

**Best for**: Learning MCP basics, simple automation tasks

```bash
create-dynemcp my-project --template default
```

**What you get**:

- Basic MCP server setup with stdio transport
- Example tool for text manipulation
- Sample resource with static content
- Simple prompt for AI assistance
- TypeScript configuration
- Build scripts and hot reload

**Project Structure**:

```
my-project/
├── src/
│   ├── index.ts              # Main entry point
│   ├── tools/                # Example tools
│   │   └── tools.ts
│   ├── resources/            # Example resources
│   │   └── resource.ts
│   └── prompts/             # Example prompts
│       └── prompt.ts
├── dynemcp.config.json      # Basic configuration
├── package.json
├── tsconfig.json
└── README.md
```

## Post-Generation Steps

After creating your project:

1. **Navigate to the project directory**:

   ```bash
   cd my-project
   ```

2. **Install dependencies**:

   ```bash
   npm install
   ```

3. **Start development server**:

   ```bash
   npm run dev
   ```

4. **Build for production**:

   ```bash
   npm run build
   ```

5. **Test your server**:
   ```bash
   npm test
   ```

## Customizing Templates

### Adding New Tools

Create new tools in the `src/tools` directory:

```typescript
// src/tools/my-custom-tool.ts
import { z } from 'zod'
import { ToolDefinition } from '@dynemcp/dynemcp'

const myCustomTool: ToolDefinition = {
  name: 'my-custom-tool',
  description: 'Does something amazing',
  schema: z.object({
    input: z.string().describe('Input for processing'),
  }),
  handler: async ({ input }) => {
    return { result: `Processed: ${input}` }
  },
}

export default myCustomTool
```

### Adding Resources

Create new resources in the `src/resources` directory:

```typescript
// src/resources/my-custom-resource.ts
import { ResourceDefinition } from '@dynemcp/dynemcp'

const myCustomResource: ResourceDefinition = {
  uri: 'custom://resource',
  name: 'Custom Resource',
  content: 'Custom content',
  contentType: 'text/plain',
}

export default myCustomResource
```

### Adding Prompts

Create new prompts in the `src/prompts` directory:

```typescript
// src/prompts/my-custom-prompt.ts
import { PromptDefinition } from '@dynemcp/dynemcp'

const myCustomPrompt: PromptDefinition = {
  id: 'custom-prompt',
  name: 'Custom Prompt',
  content: 'Custom prompt content',
}

export default myCustomPrompt
```
