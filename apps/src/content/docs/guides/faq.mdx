# Frequently Asked Questions

## General Questions

### What is DyneMCP?

DyneMCP is a TypeScript framework for building Model Context Protocol (MCP) servers. It provides a structured way to create, manage, and deploy AI tools and resources.

### Why should I use DyneMCP?

DyneMCP offers:

- Type-safe tool development
- Multiple transport protocols
- Built-in security features
- Performance optimization
- Easy deployment options
- Extensive testing utilities

### What are the system requirements?

- Node.js v18 or later
- npm or yarn
- TypeScript 4.7+
- 512MB RAM minimum
- 1GB disk space recommended

## Getting Started

### How do I create a new project?

```bash
# Using npm
npm create dynemcp@latest my-project

# Using yarn
yarn create dynemcp my-project

# Follow the prompts to select a template
```

### Which template should I choose?

1. **default**: Basic setup for simple tools
2. **calculator**: Example with basic math operations
3. **http-server**: HTTP-based server setup
4. **secure-agent**: Enhanced security features
5. **dynamic-agent**: Dynamic tool loading

Choose based on your needs:

- Start with `default` for learning
- Use `calculator` for examples
- Pick `http-server` for web APIs
- Select `secure-agent` for production
- Choose `dynamic-agent` for flexibility

### How do I run the development server?

```bash
# Start development server
npm run dev

# With specific port
PORT=3000 npm run dev

# With HTTPS
HTTPS=true npm run dev
```

## Tools and Components

### How do I create a new tool?

```typescript
import { DyneMCPTool } from '@dynemcp/dynemcp'
import { z } from 'zod'

export class MyTool extends DyneMCPTool {
  get name() {
    return 'my-tool'
  }

  readonly description = 'My custom tool'

  readonly schema = z.object({
    input: z.string(),
  })

  async execute(params: any) {
    return `Processed: ${params.input}`
  }
}
```

### How do I handle tool dependencies?

```typescript
export class AdvancedTool extends DyneMCPTool {
  async execute(params: any) {
    // Use another tool
    const basicTool = this.server.getTool('basic')
    const result = await basicTool.execute(params)

    // Use a resource
    const data = await this.server.getResource('data://config')

    return {
      result,
      config: data,
    }
  }
}
```

### How do I validate tool input?

```typescript
readonly schema = z.object({
  // String validation
  name: z.string()
    .min(3, 'Name too short')
    .max(50, 'Name too long'),

  // Number validation
  age: z.number()
    .int('Must be integer')
    .min(0, 'Must be positive'),

  // Optional fields
  email: z.string()
    .email('Invalid email')
    .optional(),

  // Enums
  role: z.enum(['user', 'admin']),

  // Arrays
  tags: z.array(z.string())
    .nonempty('At least one tag required')
})
```

## Security

### How do I secure my server?

1. Enable authentication:

```typescript
const server = createMCPServer({
  security: {
    authentication: {
      type: 'bearer',
      secret: process.env.JWT_SECRET,
    },
  },
})
```

2. Configure CORS:

```typescript
const server = createMCPServer({
  security: {
    cors: {
      origin: ['https://app.example.com'],
      methods: ['GET', 'POST'],
    },
  },
})
```

3. Set rate limits:

```typescript
const server = createMCPServer({
  security: {
    rateLimit: {
      enabled: true,
      maxRequests: 100,
      windowMs: 900000,
    },
  },
})
```

### How do I handle sensitive data?

1. Use environment variables:

```typescript
// .env
API_KEY=your-secret-key
DATABASE_URL=postgresql://user:pass@host/db

// server.ts
const apiKey = process.env.API_KEY
```

2. Secure storage:

```typescript
export class SecureStorage {
  async store(key: string, value: string) {
    // Encrypt before storing
    const encrypted = await encrypt(value)
    await this.save(key, encrypted)
  }

  async retrieve(key: string) {
    // Decrypt after retrieving
    const encrypted = await this.load(key)
    return decrypt(encrypted)
  }
}
```

## Performance

### How do I optimize performance?

1. Enable caching:

```typescript
const server = createMCPServer({
  performance: {
    caching: {
      enabled: true,
      ttl: 3600,
      maxSize: 1000,
    },
  },
})
```

2. Use connection pooling:

```typescript
const pool = new Pool({
  max: 20,
  idleTimeoutMillis: 30000,
})

export class DatabaseTool extends DyneMCPTool {
  async execute(params: any) {
    const client = await pool.connect()
    try {
      return await client.query(params.query)
    } finally {
      client.release()
    }
  }
}
```

3. Implement rate limiting:

```typescript
const server = createMCPServer({
  performance: {
    rateLimit: {
      enabled: true,
      maxRequests: 100,
      windowMs: 900000,
    },
  },
})
```

### How do I monitor performance?

1. Enable metrics:

```typescript
const server = createMCPServer({
  performance: {
    metrics: {
      enabled: true,
      interval: 5000,
    },
  },
})
```

2. Use monitoring:

```typescript
server.on('request', (req, res) => {
  const start = Date.now()

  res.on('finish', () => {
    const duration = Date.now() - start
    metrics.record('request_duration', duration)
  })
})
```

## Deployment

### How do I deploy to production?

1. Docker:

```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY . .
RUN npm ci --production
CMD ["npm", "start"]
```

2. Kubernetes:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: dynemcp
spec:
  replicas: 3
  template:
    spec:
      containers:
        - name: dynemcp
          image: dynemcp:latest
          env:
            - name: NODE_ENV
              value: production
```

3. Serverless:

```typescript
export const handler = async (event: any) => {
  const server = createMCPServer()

  try {
    const result = await server.execute(event.tool, event.params)

    return {
      statusCode: 200,
      body: JSON.stringify(result),
    }
  } catch (error) {
    return {
      statusCode: 500,
      body: JSON.stringify({ error: error.message }),
    }
  }
}
```

## Troubleshooting

### Common Errors

1. Connection Issues:

```typescript
server.on('error', (error) => {
  if (error.code === 'ECONNREFUSED') {
    // Retry connection
    setTimeout(() => server.reconnect(), 5000)
  }
})
```

2. Memory Issues:

```typescript
const usage = process.memoryUsage()
if (usage.heapUsed > threshold) {
  // Clean up resources
  gc()
  // Notify admin
  notify('High memory usage')
}
```

3. Timeout Issues:

```typescript
export class TimeoutTool extends DyneMCPTool {
  async execute(params: any) {
    return Promise.race([
      this.operation(params),
      new Promise((_, reject) =>
        setTimeout(() => reject(new Error('Timeout')), 5000)
      ),
    ])
  }
}
```

### Debug Mode

Enable debug mode:

```typescript
const server = createMCPServer({
  debug: {
    enabled: true,
    verbose: true,
    logLevel: 'debug',
  },
})
```

Use debug logging:

```typescript
server.on('request', (req) => {
  debug('Request:', {
    tool: req.tool,
    params: req.params,
    timestamp: new Date(),
  })
})
```

## Updates and Migration

### How do I update DyneMCP?

1. Check current version:

```bash
npm list @dynemcp/dynemcp
```

2. Update packages:

```bash
npm update @dynemcp/dynemcp
```

3. Check for breaking changes:

```bash
npm run typecheck
npm test
```

### How do I migrate between versions?

1. Read changelog:

```bash
# View changelog
npm run changelog
```

2. Update code:

```typescript
// Before (v1.0)
server.use(tool)

// After (v2.0)
server.register(tool)
```

3. Run migration:

```bash
npm run migrate
```
