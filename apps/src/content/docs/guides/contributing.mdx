# Contributing Guide

This guide covers how to contribute to the DyneMCP framework, from setting up your development environment to submitting pull requests.

## Getting Started

### Prerequisites

1. Node.js (v18 or later)
2. npm or yarn
3. Git
4. TypeScript knowledge
5. Testing experience

### Development Setup

```bash
# Clone repository
git clone https://github.com/dynemcp/dynemcp.git
cd dynemcp

# Install dependencies
npm install

# Build project
npm run build

# Run tests
npm test
```

## Development Workflow

### 1. Create Branch

```bash
# Update main
git checkout main
git pull origin main

# Create feature branch
git checkout -b feature/my-feature
```

### 2. Make Changes

```typescript
// Add new feature
export class NewFeature extends DyneMCPTool {
  get name() {
    return 'new-feature'
  }

  readonly description = 'A new feature'

  readonly schema = z.object({
    input: z.string(),
  })

  async execute(params: any) {
    // Implementation
  }
}
```

### 3. Write Tests

```typescript
describe('NewFeature', () => {
  let tool: NewFeature

  beforeEach(() => {
    tool = new NewFeature()
  })

  test('processes input correctly', async () => {
    const result = await tool.execute({
      input: 'test',
    })

    expect(result).toBeDefined()
  })

  test('validates input', async () => {
    await expect(
      tool.execute({
        input: 123,
      })
    ).rejects.toThrow()
  })
})
```

## Code Style

### TypeScript Guidelines

```typescript
// Use interfaces for public APIs
interface FeatureOptions {
  enabled: boolean
  timeout: number
}

// Use type for internal types
type FeatureResult = {
  success: boolean
  data: unknown
}

// Use classes for components
export class Feature {
  // Private fields with #
  #options: FeatureOptions

  // Constructor with validation
  constructor(options: FeatureOptions) {
    this.#validateOptions(options)
    this.#options = options
  }

  // Public methods with JSDoc
  /**
   * Process input data
   * @param input Input data
   * @returns Processing result
   */
  async process(input: string): Promise<FeatureResult> {
    // Implementation
  }

  // Private methods
  #validateOptions(options: FeatureOptions) {
    if (options.timeout < 0) {
      throw new Error('Invalid timeout')
    }
  }
}
```

### Naming Conventions

```typescript
// File names: kebab-case
// feature-manager.ts

// Classes: PascalCase
class FeatureManager {}

// Interfaces: PascalCase with I prefix
interface IFeatureOptions {}

// Types: PascalCase
type FeatureResult = {}

// Variables/Functions: camelCase
const featureInstance = new Feature()
function processFeature() {}

// Constants: UPPER_SNAKE_CASE
const MAX_RETRIES = 3
```

## Testing Guidelines

### Unit Tests

```typescript
describe('Feature', () => {
  // Setup/Teardown
  beforeEach(() => {
    // Setup
  })

  afterEach(() => {
    // Cleanup
  })

  // Group related tests
  describe('validation', () => {
    test('validates options', () => {
      // Test
    })

    test('handles invalid input', () => {
      // Test
    })
  })

  // Use clear descriptions
  test('processes data correctly', () => {
    // Arrange
    const input = 'test'
    const expected = 'result'

    // Act
    const result = feature.process(input)

    // Assert
    expect(result).toBe(expected)
  })
})
```

### Integration Tests

```typescript
describe('Feature Integration', () => {
  let server: TestServer

  beforeAll(async () => {
    server = await TestServer.create()
  })

  afterAll(async () => {
    await server.stop()
  })

  test('integrates with other components', async () => {
    // Test complete flow
    const result = await server.execute('feature', { input: 'test' })

    expect(result).toMatchObject({
      success: true,
      data: expect.any(Object),
    })
  })
})
```

## Documentation Guidelines

### API Documentation

```typescript
/**
 * Feature for processing data
 * @class
 */
export class Feature {
  /**
   * Create a new feature
   * @param {FeatureOptions} options - Configuration options
   * @throws {Error} If options are invalid
   */
  constructor(options: FeatureOptions) {}

  /**
   * Process input data
   * @param {string} input - Input data
   * @returns {Promise<FeatureResult>} Processing result
   * @throws {Error} If processing fails
   */
  async process(input: string): Promise<FeatureResult> {}
}
```

### Example Documentation

```typescript
// Basic usage
const feature = new Feature({
  enabled: true,
  timeout: 1000,
})

// With error handling
try {
  const result = await feature.process('input')
  console.log('Success:', result)
} catch (error) {
  console.error('Failed:', error)
}

// Advanced usage
const feature = new Feature({
  enabled: true,
  timeout: 1000,
  onError: (error) => {
    // Handle error
  },
})

await feature.process('input', {
  retry: true,
  maxRetries: 3,
})
```

## Pull Request Process

### 1. Preparation

- Update your branch with main
- Run all tests
- Update documentation
- Check code style
- Review changes

### 2. Pull Request Template

```markdown
## Description

Brief description of changes

## Type of Change

- [ ] Bug fix
- [ ] New feature
- [ ] Breaking change
- [ ] Documentation update

## Testing

- [ ] Unit tests added/updated
- [ ] Integration tests added/updated
- [ ] Manual testing performed

## Documentation

- [ ] API documentation updated
- [ ] Examples added/updated
- [ ] README updated
- [ ] CHANGELOG updated

## Additional Notes

Any additional information
```

### 3. Review Process

1. Code Review

   - Style consistency
   - Test coverage
   - Documentation
   - Performance
   - Security

2. CI/CD Checks

   - Build success
   - Tests passing
   - Linting
   - Type checking

3. Documentation Review
   - API docs complete
   - Examples accurate
   - CHANGELOG updated
