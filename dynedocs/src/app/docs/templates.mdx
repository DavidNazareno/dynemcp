# Templates

DyneMCP provides several pre-built templates to help you get started quickly. Each template is designed for specific use cases and includes everything you need to build and deploy your MCP server.

## Available Templates

### Default Template

**Best for**: Learning MCP basics, simple automation, prototyping

The default template provides a minimal but complete MCP server setup with examples of all core components.

```bash
create-dynemcp my-project --template default
```

**What's included**:

- Basic tool example (text manipulation)
- Simple resource (static content)
- Example prompt (AI assistance)
- Complete TypeScript setup
- Build and development scripts

**Project Structure**:

```
my-project/
├── src/
│   ├── tools/
│   │   └── text-tool.ts        # Basic text manipulation
│   ├── resources/
│   │   └── help-resource.ts    # Static help content
│   ├── prompts/
│   │   └── assistant-prompt.ts # General assistant prompt
│   └── index.ts                # Server entry point
├── dynemcp.config.json         # Basic configuration
└── package.json                # Dependencies and scripts
```

**Use Cases**:

- Learning MCP concepts
- Rapid prototyping
- Building simple automation tools
- Educational purposes

### Calculator Template

**Best for**: Mathematical computations, educational tools, scientific applications

A comprehensive mathematical agent with basic and advanced calculation capabilities.

```bash
create-dynemcp my-calculator --template calculator
```

**What's included**:

- Basic arithmetic operations (add, subtract, multiply, divide)
- Advanced mathematical functions (trigonometry, logarithms, statistics)
- Mathematical constants and formulas resource
- Specialized mathematical problem-solving prompts
- Error handling for mathematical edge cases

**Project Structure**:

```
my-calculator/
├── src/
│   ├── tools/
│   │   ├── basic-calculator.ts     # Basic arithmetic
│   │   └── advanced-calculator.ts  # Advanced functions
│   ├── resources/
│   │   └── math-reference.ts       # Mathematical formulas
│   ├── prompts/
│   │   └── calculator-prompt.ts    # Math problem solving
│   └── index.ts
├── dynemcp.config.json
└── package.json
```

**Features**:

- **Basic Operations**: Addition, subtraction, multiplication, division
- **Advanced Functions**: sin, cos, tan, log, exp, sqrt, power
- **Statistics**: mean, median, mode, standard deviation
- **Unit Conversion**: length, weight, temperature, currency
- **Mathematical Constants**: π, e, golden ratio, etc.

### HTTP Server Template

**Best for**: Web integration, API endpoints, microservices

A web-based MCP server using HTTP transport, perfect for integration with web applications.

```bash
create-dynemcp my-server --template http-server
```

**What's included**:

- Express.js HTTP server setup
- RESTful endpoint configuration
- CORS middleware for cross-origin requests
- HTTP transport for MCP communication
- Basic greeting and server info tools
- Health check endpoints

**Project Structure**:

```
my-server/
├── src/
│   ├── tools/
│   │   └── greet.ts              # HTTP greeting tool
│   ├── resources/
│   │   └── server-info.ts        # Server status info
│   ├── prompts/
│   │   └── introduction.ts       # API introduction
│   └── index.ts                  # HTTP server setup
├── dynemcp.config.json           # HTTP transport config
└── package.json
```

**Configuration**:

```json
{
  "transport": {
    "type": "http",
    "port": 3000,
    "host": "localhost",
    "cors": true,
    "middleware": ["compression", "helmet"]
  }
}
```

**Features**:

- **HTTP Transport**: RESTful API endpoints
- **CORS Support**: Cross-origin resource sharing
- **Middleware**: Compression, security headers, logging
- **Health Checks**: Server status monitoring
- **API Documentation**: Auto-generated OpenAPI specs

A production-ready MCP server with comprehensive security features and enterprise-grade capabilities.

```bash
create-dynemcp my-secure-agent --template secure-agent
```

**What's included**:

- API key authentication middleware
- HTTP-Stream transport with session management
- Rate limiting and CORS configuration
- Security headers and input validation
- Audit logging capabilities
- Production-ready configuration
- Environment-based configuration

**Security Features**:

- **Authentication**: API key, JWT, OAuth2 support
- **Rate Limiting**: Configurable per-user and global limits
- **Input Validation**: Comprehensive request validation
- **Audit Logging**: Security event tracking
- **HTTPS**: TLS/SSL encryption
- **CORS**: Configurable cross-origin policies
- **Security Headers**: Helmet.js integration

## Customizing Templates

### Adding Custom Tools

After creating a project, you can add new tools:

```bash
cd my-project
dynemcp add tool my-custom-tool
```

### Modifying Configuration

Each template comes with a `dynemcp.config.json` that you can customize:

```json
{
  "name": "my-customized-server",
  "version": "1.0.0",
  "description": "My customized MCP server",
  "transport": {
    "type": "stdio"
  },
  "build": {
    "outDir": "dist",
    "bundle": true,
    "minify": true
  },
  "security": {
    "auth": {
      "type": "none"
    }
  }
}
```

### Template Inheritance

You can extend existing templates by creating a custom template:

```bash
# Create custom template based on calculator
dynemcp create-template my-math-server --extends calculator
```

## Best Practices

### Choosing the Right Template

1. **Start Simple**: Use the default template for learning
2. **Match Use Case**: Choose templates that align with your goals
3. **Consider Scale**: Pick templates appropriate for your deployment scale
4. **Security First**: Use secure-agent for production environments

### Template Modification

1. **Understand Structure**: Familiarize yourself with the template structure
2. **Incremental Changes**: Make small, incremental modifications
3. **Test Thoroughly**: Test after each significant change
4. **Document Changes**: Keep track of your customizations

### Development Workflow

1. **Create from Template**: Start with the closest matching template
2. **Customize Configuration**: Modify `dynemcp.config.json` as needed
3. **Add Components**: Use `dynemcp add` to add tools/resources/prompts
4. **Test Locally**: Use `dynemcp dev` for development
5. **Build and Deploy**: Use `dynemcp build` for production

## Creating Custom Templates

### Template Structure

```
my-template/
├── template.json           # Template metadata
├── src/
│   ├── index.ts.template   # Template files
│   └── tools/
├── dynemcp.config.json.template
└── package.json.template
```

### Template Variables

Use variables in your templates:

```json
{
  "name": "{{project-name}}",
  "version": "{{version}}",
  "description": "{{description}}"
}
```

### Publishing Templates

```bash
# Package your template
dynemcp template pack my-template

# Publish to npm
npm publish my-dynemcp-template
```
